# GitHub ゼロベース体験 — 全体計画書

## 概要

空のディレクトリから GitHub リポジトリの作成・コード開発・更新・参照までを
一通り体験するハンズオンです。

---

## 前提条件

| 項目 | 必要なもの |
|------|-----------|
| Git | インストール済みであること |
| GitHub CLI (`gh`) | インストール済み & `gh auth login` 済みであること |
| Node.js | インストール済みであること |
| GitHub アカウント | 作成済みであること |

---

## ステップ一覧

| Step | タイトル | 概要 |
|------|---------|------|
| 0 | 環境確認 | 必要なツールが使えるか確認 |
| 1 | Git 初期化 | ローカルリポジトリを作成 |
| 2 | サンプルコード作成 | Node.js の簡単なアプリを作成 |
| 3 | 初回コミット | コードを Git に記録 |
| 4 | GitHub リポジトリ作成 & プッシュ | リモートリポジトリを作成して公開 |
| 5 | コード更新 | 機能を追加してコミット & プッシュ |
| 6 | リポジトリ参照 & コード追加 | GitHub 上の情報を参照して追加開発 |
| 7 | 確認 & クリーンアップ | 最終確認 |

---

## 各ステップの詳細

### Step 0: 環境確認

**目的**: 必要なツールがすべて使える状態か確認する

**作業内容**:
- `git --version` で Git の確認
- `gh auth status` で GitHub CLI の認証確認
- `node --version` で Node.js の確認

**成果物**: なし（確認のみ）

---

### Step 1: Git 初期化

**目的**: 現在のディレクトリを Git リポジトリにする

**作業内容**:
- `git init` を実行
- `.gitignore` ファイルを作成（node_modules 等を除外）

**成果物**:
- `.git/` ディレクトリ（Git 管理情報）
- `.gitignore`

---

### Step 2: サンプルコード作成

**目的**: GitHub にプッシュするためのサンプルアプリを作成する

**作業内容**:
- `npm init -y` で package.json を生成
- `index.js` — メインのアプリケーション（簡単な TODO 管理ツール）
- `README.md` — プロジェクトの説明

**作成するアプリの仕様**:
- コマンドラインで動く TODO リスト管理ツール
- 機能: TODO の追加・一覧表示
- シンプルで理解しやすいコード

**成果物**:
- `package.json`
- `index.js`
- `README.md`

---

### Step 3: 初回コミット

**目的**: 作成したコードを Git に記録する

**作業内容**:
- `git add .` で全ファイルをステージング
- `git commit -m "Initial commit: TODO管理ツール"` でコミット
- `git log` でコミット履歴を確認

**成果物**: Git の初回コミット

---

### Step 4: GitHub リポジトリ作成 & プッシュ

**目的**: GitHub 上にリモートリポジトリを作成し、コードを公開する

**作業内容**:
- `gh repo create cc-mcp-test01 --public --source=. --push` を実行
  - GitHub にリポジトリを新規作成
  - ローカルのコードをプッシュ
- ブラウザで GitHub 上のリポジトリを確認

**成果物**: GitHub 上の公開リポジトリ

---

### Step 5: コード更新

**目的**: コードに機能を追加し、変更を GitHub に反映する

**作業内容**:
- `index.js` に新機能を追加（TODO の削除機能）
- `git add` → `git commit` → `git push` の一連の流れを実行
- GitHub 上で変更が反映されたことを確認

**成果物**: 更新されたコード & GitHub 上の2つ目のコミット

---

### Step 6: リポジトリ参照 & コード追加

**目的**: GitHub 上のリポジトリ情報を参照し、それを元にコードを追加する

**作業内容**:
- `gh repo view` でリポジトリ情報を取得
- `gh api` でリポジトリのファイル一覧を取得
- 取得した情報を元に新しいファイル（`utils.js`）を追加
- コミット & プッシュ

**成果物**: GitHub から参照した情報を元に追加されたコード

---

### Step 7: 確認 & クリーンアップ

**目的**: 全体の成果を確認する

**作業内容**:
- `git log --oneline` で全コミット履歴を確認
- `gh repo view --web` でブラウザからリポジトリを確認
- 体験の振り返り

**成果物**: 完成したリポジトリ

---

## 最終的なファイル構成（予定）

```
cc_mcp_test01/
├── .gitignore
├── package.json
├── README.md
├── index.js        # メインアプリ（TODO管理ツール）
└── utils.js        # ユーティリティ関数
```

## 所要時間の目安

各ステップは Claude Code が自動で実行します。
ユーザーの確認を含めて全体で 10〜15 分程度です。
